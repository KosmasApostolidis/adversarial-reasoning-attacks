## Risks + Mitigations

| Risk | Mitigation |
|------|------------|
| Rebase conflict (main moved since merge-tree check) | Re-run `git merge-tree` after `git fetch`; if conflicts surface, abort rebase and resolve manually |
| Pre-commit hook reformats version-bump commit | Re-stage with `git add` and re-commit (never `--amend`, never `--no-verify`) |
| CI red on PR | `gh pr checks <PR#> --watch`; fix root cause on branch, push, re-run; ask user if not resolved on first attempt |
| Race: someone pushes to main between rebase and merge | `gh pr merge` auto-rebases or fails; on fail, `git pull --rebase`, force-push, retry |
| Tag `v0.2.0` already exists on remote | Pre-check `git ls-remote --tags origin v0.2.0`; abort and ask if present |
| `--generate-notes` thin (no prior tag) | Append hand-written changelog via `gh release edit v0.2.0 --notes-file` |
| Local branch persists after `gh pr merge --delete-branch` | Explicit `git branch -D fix/round4-logic-bugs` step |
| Force-push lease fails (concurrent push) | `git fetch`, inspect, retry only if state is safe |
